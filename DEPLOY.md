# Инструкция по деплою на ringova.com

## Текущая конфигурация

Сайт использует гибридный билд:
- **Astro** — лендинг (`/`) и услуги (`/services`)
- **Hugo** — документация (`/docs`, `/samples`)

## Варианты деплоя

### 1. Netlify (рекомендуется)

**Настройки в Netlify:**
- Build command: `chmod +x build.sh && ./build.sh`
- Publish directory: `public`
- Node version: `20`
- Hugo version: `0.132.2`

**Домен:**
- В настройках Netlify добавьте кастомный домен `ringova.com`
- Файл `CNAME` уже есть в репозитории

**Автоматический деплой:**
- При пуше в `main` Netlify автоматически запустит билд

### 2. GitHub Pages

**Настройки:**
- GitHub Actions workflow уже обновлён (`.github/workflows/pages.yaml`)
- При пуше в `main` автоматически задеплоится

**Домен:**
- В настройках GitHub Pages укажите кастомный домен `ringova.com`
- Файл `CNAME` уже есть в репозитории

### 3. Ручной деплой

```bash
# 1. Установите зависимости (если ещё не установлены)
npm install

# 2. Запустите билд
./build.sh

# 3. Загрузите содержимое папки public/ на ваш хостинг
```

## Проверка перед деплоем

1. Убедитесь, что билд проходит локально:
   ```bash
   ./build.sh
   ```

2. Проверьте содержимое `public/`:
   - Должны быть `index.html` и `services/index.html` (от Astro)
   - Должны быть `docs/`, `samples/` (от Hugo)

3. Проверьте, что все стили и скрипты загружаются

## После деплоя

1. Проверьте главную страницу — должна быть от Astro
2. Проверьте `/services` — должна быть от Astro
3. Проверьте `/docs` — должна быть от Hugo
4. Проверьте, что все ссылки работают

## Откат (если что-то пошло не так)

Если нужно вернуться к старому варианту (только Hugo):
1. В Netlify/GitHub Pages измените build command на: `hugo --gc --minify`
2. Или откатите коммит в git
